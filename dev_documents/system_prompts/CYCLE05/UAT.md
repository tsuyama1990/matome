# Cycle 05: User Acceptance Testing (UAT)

## 1. Test Scenarios

These scenarios verify the correctness of the **Semantic Zooming & Polish**.

### Scenario 1: Navigation (Semantic Zooming)
**Priority**: Critical
**Goal**: Ensure users can seamlessly traverse the DIKW hierarchy.
- **Steps**:
  1. Launch the `MatomeCanvas` application.
  2. Verify the initial view shows the "Wisdom" (L1) node.
  3. Click on the "Knowledge" (L2) children displayed below.
  4. Verify the view updates to show the selected L2 node as the focus.
  5. Verify breadcrumbs update to reflect the path (e.g., "Root > Knowledge 1").
  6. Click a breadcrumb or "Back" button to return to Root.

## 2. Behavior Definitions (Gherkin)

### Feature: Interactive Navigation

**Scenario: Drill Down**
  GIVEN I am viewing a parent node `P`
  WHEN I click on a child node card `C`
  THEN the main view should update to display `C` as the current node
  AND `C` should be added to the navigation history (breadcrumbs).

**Scenario: Navigate Up**
  GIVEN I am deep in the hierarchy at node `C`
  WHEN I click the "Parent" link or Breadcrumb
  THEN the main view should return to the parent node `P`.

### Feature: Refinement

**Scenario: Successful Refinement**
  GIVEN I am viewing a node
  WHEN I submit a refinement instruction via the chat input
  THEN the UI should show a "Processing..." state
  AND upon completion, the node's text should update to the new version generated by the engine
  AND the chat input should be cleared or show a success message.
